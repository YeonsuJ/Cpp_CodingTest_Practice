**DP(동적 계획법) 문제 풀이의 90%는 '점화식(규칙)을 찾아내는 것'**이고, 나머지 10%가 그걸 코드로 옮기는 작업(구현)입니다.그래서 DP 문제는 "코딩 실력"보다는 "수학적 귀납법"이나 "규칙 찾기 능력"을 테스트하는 쪽에 가깝습니다.💡 DP 문제를 푼다는 것의 진짜 과정보통 DP 문제를 만나면 다음과 같은 3단계 과정을 거치게 됩니다.1단계: 상태 정의 (가장 중요!)배열 dp[i]가 무엇을 의미하는지 말로 정의해야 합니다. 이걸 잘못 정하면 점화식을 절대 못 세웁니다.1003번 문제 예시: dp[i] = "$N=i$일 때 0이 출력되는 횟수"2단계: 점화식 구하기 (연수님이 말씀하신 부분)현재의 답(dp[i])을 구하기 위해 이전의 답들(dp[i-1], dp[i-2] 등)을 어떻게 지지고 볶아야 하는지 관계식을 찾습니다.1003번 문제 예시: dp[i] = dp[i-1] + dp[i-2] (피보나치 규칙)3단계: 초기값(Base Case) 설정점화식이 작동하기 시작하는 맨 처음 도미노를 세워줍니다.1003번 문제 예시: dp[0] = 1, dp[1] = 0🚀 코딩테스트 실전 팁DP 문제가 어렵게 느껴지는 이유는 점화식이 눈에 바로 보이지 않기 때문입니다. 실전에서는 이렇게 접근해보세요.손으로 써보기 (노가다):머리로만 생각하지 말고 $N=1, N=2, N=3, N=4 \dots$ 일 때의 정답을 손으로 직접 구해서 적어보세요.규칙 발견:적어둔 숫자들 사이의 관계를 째려봅니다."어? 앞의 두 개를 더하니까 이게 나오네?" (피보나치 유형)"어? 전 단계 값에 2를 곱하고 1을 더하면 되네?""어? 전 단계의 값들 중 최솟값에 현재 값을 더하면 되네?"식으로 옮기기:발견한 규칙을 dp[i] = ... 형태의 수식으로 옮기면 끝입니다.결론:연수님이 "DP는 점화식을 구하는 거야?"라고 느끼셨다면, 이미 DP의 핵심을 꿰뚫으신 겁니다. 코드는 그저 그 식을 컴퓨터에게 알려주는 수단일 뿐이니까요. 자신감을 가지셔도 좋습니다!
