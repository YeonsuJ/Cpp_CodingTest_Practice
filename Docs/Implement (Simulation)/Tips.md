C++ 사용자가 알아야 할 '구현/시뮬레이션' 핵심 팁
작성자님의 목표(실버 상위 ~ Lv2/3)에서 구현/시뮬레이션 문제는 **알고리즘적 사고력보다는 '피지컬(코딩 지구력)'**을 테스트하는 경우가 많습니다.

C++로 이 유형을 풀 때 가장 중요한 3가지 테크닉입니다.

① 방향 벡터 (dx, dy) 필수 암기
2차원 배열(격자)에서 상하좌우로 이동하는 시뮬레이션이 90%입니다. 이 코드는 숨 쉬듯이 나와야 합니다.

C++
```
// 상, 하, 좌, 우 (문제 좌표계에 따라 다름)
int dx[4] = {-1, 1, 0, 0};
int dy[4] = {0, 0, -1, 1};

for(int i=0; i<4; i++) {
    int nx = x + dx[i];
    int ny = y + dy[i];
    // 범위 체크 (필수!)
    if(nx < 0 || nx >= N || ny < 0 || ny >= M) continue;
    // 이동 로직...
}
```

② 문제는 '그대로' 읽기 (자의적 해석 금지)
구현 문제는 지문이 깁니다.

"A를 한 후 B를 한다"와 "A와 B를 동시에 한다"는 천지 차이입니다.

지문에 적힌 순서(1번 → 2번 → 3번)를 주석으로 달아놓고 코딩을 시작하세요.

③ 디버깅이 생명
C++은 Python처럼 중간 출력이 쉽지 않습니다. 시뮬레이션은 중간에 한 단계만 삐끗해도 결과가 완전히 달라집니다.

배열이 제대로 회전했는지, 로봇이 제대로 이동했는지 단계별로 cout을 찍어 확인하는 습관을 들이세요.

요약
구현은 "시키는 대로 짜는 것"의 총칭.

시뮬레이션은 "단계별로 상황을 진행시키는" 구현의 심화 버전.

백준이나 프로그래머스에서는 **굳이 구분하지 말고 "구현"**이라고 생각하고 푸시면 됩니다.
